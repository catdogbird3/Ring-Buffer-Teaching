📚 什麼是 Ring Buffer？

🎯 核心概念

Ring Buffer（環形緩衝區）是一種固定大小的資料結構， 當寫入到尾端時會繞回到開頭繼續寫入，形成一個「環」。

⚡ 為什麼用它？

  • ✓ 固定記憶體：不需要動態分配，避免記憶體碎片
  • ✓ O(1) 操作：讀寫都是常數時間
  • ✓ 生產者-消費者：完美處理不同速度的資料流

💡 常見應用

UART 接收緩衝 音訊串流 網路封包 日誌記錄 鍵盤輸入

// 簡化程式碼

write_idx = (write_idx + 1) % SIZE;
read_idx = (read_idx + 1) % SIZE;
// 透過取餘數實現「環繞」效果

📚 UART 說明

🎯 什麼是 UART？

Universal Asynchronous Receiver/Transmitter
通用非同步收發器，最簡單的串列通訊協議，只需要 2 條線（TX 發送、RX 接收）。

⚙️ 關鍵參數

鮑率
9600 ~ 115200
資料位元
8 bits
停止位元
1 bit
校驗位元
None/Even/Odd

✨ 特點

  • • 非同步：不需要時鐘線，雙方約定好鮑率
  • • 點對點：一對一通訊
  • • 全雙工：可同時收發
  • • 簡單：最容易實現的協議
Debug Console GPS 模組 藍牙模組 Arduino Serial

📚 I2C 說明

🎯 什麼是 I2C？

Inter-Integrated Circuit
同步、多主從架構的串列通訊協議，只需要 2 條線就能連接多達 127 個設備！

🔌 接線

SCL
時鐘線
SDA
資料線

✨ 特點

  • • 同步：有時鐘線，Master 控制節奏
  • • 多從設備：用位址區分設備
  • • 應答機制：ACK/NACK 確認
  • • 半雙工：同一時間只能單向
OLED 螢幕 溫濕度感測器 EEPROM 加速度計 RTC 時鐘

⚡ SPI 連接架構

全雙工傳輸 - MOSI 和 MISO 同時工作

📚 SPI 說明

🎯 什麼是 SPI？

Serial Peripheral Interface
高速同步串列通訊協議，需要 4 條線，但換來超高速度和全雙工能力！

🔌 四條信號線

SCK (SCLK)
時鐘 - Master 產生
MOSI
Master Out, Slave In
MISO
Master In, Slave Out
CS/SS
晶片選擇 (低電位啟用)

✨ 特點

  • • 超高速：可達數十 MHz
  • • 全雙工：同時收發資料
  • • 簡單協議：無位址、無應答
  • • 多 CS：每個從設備一條 CS 線
SD 卡 TFT 螢幕 Flash 記憶體 ADC/DAC

📚 I2S 說明

🎯 什麼是 I2S？

Inter-IC Sound
專為數位音訊設計的串列通訊協議，用 3 條線傳輸高品質立體聲音訊。

🔌 三條信號線

SCK (BCLK) 位元時鐘
WS (LRCLK) 字選擇/左右聲道
SD 串列資料

✨ 特點

  • • 音訊專用：為音訊優化的時序
  • • 立體聲：WS 切換左右聲道
  • • 高保真：支援 16/24/32 bit
  • • 低延遲：適合即時音訊
DAC 晶片 數位麥克風 藍牙音訊 ESP32 音訊

📚 USB 說明

🎯 什麼是 USB？

Universal Serial Bus
通用串列匯流排，最廣泛使用的外部連接標準，支援供電、資料傳輸和熱插拔。

🔌 連接器類型

▭
Type-A
▯
Type-B
⬭
Type-C

✨ 特點

  • • 主從架構：Host 控制所有通訊
  • • 供電能力：USB-PD 最高 240W
  • • 熱插拔：即插即用
  • • 多端點：一設備多功能

📦 設備類別

HID (鍵盤滑鼠) Mass Storage CDC (序列埠) Audio Video
